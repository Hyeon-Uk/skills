# Review-Bot Patterns

Battle-tested patterns for automation agents posting to Gerrit. Codifies the bot hygiene rules from `SKILL.md`.

## 1. Idempotency

A bot polls on an interval. It will re-see the same patchset many times. Two defences:

### (a) Server-side: `omit_duplicate_comments`

```json
{ "omit_duplicate_comments": true, "comments": {...} }
```

Gerrit drops any comment that exactly matches an existing comment from the same author on the same line. Exact match = `path + line/range + side + message`. Any whitespace difference defeats it.

### (b) Client-side: skip already-reviewed patchsets

Track `(change_number, patchset_number, bot_account_id)` locally. Query before posting:

```http
GET /a/changes/{id}/revisions/{rev}/comments
```

If any comment's `author._account_id == bot` and `tag` starts with `autogenerated:review-bot`, skip.

## 2. Tagging bot comments

Always set `tag: "autogenerated:<bot-name>"` on ReviewInput **and** on each inline comment where possible. This:
- collapses bot output in the Gerrit UI (user can toggle)
- lets you filter with `has:autogenerated` in search
- allows a later "bot re-review" to identify its own prior output

Prefix convention: `autogenerated:` is reserved by Gerrit for tooling. Bot name after the colon.

## 3. Quiet mode during development

```json
{
  "notify": "NONE",
  "message": "[bot-dev] dry run - not notifying",
  "tag": "autogenerated:review-bot-dev"
}
```

`notify: NONE` skips emails entirely. `OWNER` = only the change owner. `OWNER_REVIEWERS` = owner + current reviewers. `ALL` = default (don't use for bots).

## 4. Safe label handling

Pattern: try a non-blocking vote first, fall back to comment-only on 400.

```python
try:
    await client.post_review(chg, rev, {**body, "labels": {"Code-Review": 0}})
except httpx.HTTPStatusError as e:
    if e.response.status_code == 400:
        await client.post_review(chg, rev, body)  # no labels
    else:
        raise
```

Common safe choices:
- `"Code-Review": 0` — "I looked, no opinion"
- `"Verified": -1` — only if bot has Verify permission and you actually failed something
- omit `labels` entirely — always safe

Never let the bot vote `+2` unless the project explicitly delegates that authority.

## 5. Rate limiting (429)

Exponential backoff with jitter:

```python
import random, asyncio

async def post_with_retry(fn, *args, max_tries=5):
    for attempt in range(max_tries):
        try:
            return await fn(*args)
        except httpx.HTTPStatusError as e:
            if e.response.status_code != 429 or attempt == max_tries - 1:
                raise
            delay = (2 ** attempt) + random.random()
            await asyncio.sleep(delay)
```

If you operate multiple poller workers, **stagger their start times**: a 0-60s random sleep on boot prevents thundering herd against Gerrit.

## 6. Pagination

```python
all_changes, skip = [], 0
while True:
    page = await client.query_changes(q, opts, n=200, skip=skip)
    all_changes.extend(page)
    if not page or not page[-1].get("_more_changes"):
        break
    skip += len(page)
```

`_more_changes: true` only appears on the **last** item of a page that has more.

## 7. Partial-line comments (CommentRange)

For highlighting a specific span:

```json
{
  "path": "src/foo.c",
  "range": {
    "start_line": 50, "start_character": 4,
    "end_line":   52, "end_character": 10
  },
  "message": "Consider extracting.",
  "unresolved": false
}
```

- 1-indexed lines; `end_character` is **exclusive**.
- If both `line` and `range` set, `line` must equal `range.end_line`.
- For single-line spans, `start_line == end_line`.

## 8. robot_comments with fix suggestions

Machine-generated comments go in `robot_comments` (not `comments`). They support inline fix suggestions the user can apply with one click:

```json
{
  "robot_comments": {
    "src/foo.c": [
      {
        "line": 42,
        "message": "Use strnlen instead of strlen on untrusted input.",
        "robot_id": "review-bot",
        "robot_run_id": "2026-04-21T07:59:00Z",
        "url": "https://wiki/strlen-hazards",
        "fix_suggestions": [
          {
            "description": "Replace with strnlen",
            "replacements": [
              {
                "path": "src/foo.c",
                "range": {
                  "start_line": 42, "start_character": 8,
                  "end_line":   42, "end_character": 14
                },
                "replacement": "strnlen"
              }
            ]
          }
        ]
      }
    ]
  }
}
```

`robot_id` and `robot_run_id` are mandatory on every RobotCommentInput.

## 9. Attention Set hygiene

After posting a review, you usually want to **remove yourself** from the attention set so the owner isn't blocked waiting on the bot:

```json
{
  "message": "...",
  "remove_from_attention_set": [
    { "user": "self", "reason": "automated review posted" }
  ],
  "ignore_automatic_attention_set_rules": true
}
```

`ignore_automatic_attention_set_rules: true` suppresses Gerrit's default "bot commented, add them back" behavior.

## 10. Self-identification

On bot startup, fetch and cache the bot's own account ID:

```python
me = await client.get("/a/accounts/self")
BOT_ACCOUNT_ID = me["_account_id"]
```

Use it for (a) skipping changes already reviewed by this bot, (b) de-duping comments by author, (c) logging.

## 11. Error classification

| Status | Treat as | Action |
|---|---|---|
| 400 | Input bug | Log, do NOT retry same payload |
| 403 | Auth / perm | Refresh token once, else escalate |
| 404 | Change deleted / hidden | Drop from queue |
| 409 | Stale state (new patchset) | Re-fetch and retry |
| 412 | Precondition | Drop or reconsider flow |
| 422 | Ambiguous ID | Use `Project~Branch~Change-Id` form |
| 429 | Rate limit | Backoff + retry |
| 5xx | Transient | Retry with backoff |

## 12. Dry-run mode

For testing, provide a flag that transforms `post_review` into a no-op that logs the payload:

```python
async def post_review(self, chg, rev, body):
    if self.dry_run:
        logger.info("DRY RUN review for %s/%s: %s", chg, rev, body)
        return {"dry_run": True}
    return await self._real_post(chg, rev, body)
```

Combine with a known test project so real runs never touch production changes during dev.
