# Gerrit REST API

Integration layer for anything that talks HTTP to a Gerrit server. The `gerrit_review_bot` project's poller/formatter is the canonical consumer.

## Non-negotiables (every request)

1. **Prefix with `/a/` for authenticated calls** — `GET /a/changes/?q=...`. Without it you get anonymous access (usually 403 on writes).
2. **Strip the XSSI prefix `)]}'` before JSON-parsing** — every response starts with those 4 bytes + newline. Feed directly to `json.loads` and it explodes.
3. **HTTP Basic with an auth token (not password)** — `Authorization: Basic base64(username:http_token)`. Tokens are minted in Gerrit Settings → HTTP Credentials.
4. **URL-encode path segments** — project names like `platform/core/appfw/pkgmgr-info` need `%2F`, not raw `/`.
5. **Use `pp=0` for compact JSON** in automation — default is pretty-printed.

## Canonical review-bot flow

```
poll → fetch diff → generate review → post review
```

### 1) Poll open changes

```http
GET /a/changes/?q=status:open+project:foo+-label:Verified=-1
    &o=CURRENT_REVISION&o=CURRENT_FILES&o=DETAILED_ACCOUNTS
    &n=50&pp=0
```

- `q` can repeat for batched queries (response becomes a 2D array).
- Pagination: `S=<skip>`; look at `_more_changes` on the last item.
- Match `status:open -is:wip -age:7d` to avoid churn on drafts/stale.

### 2) Fetch the diff

Prefer the per-file DiffInfo for structured analysis:

```http
GET /a/changes/{id}/revisions/current/files               # FileInfo map
GET /a/changes/{id}/revisions/current/diff/{path}?pp=0    # DiffInfo
```

Or grab the unified patch (Base64):

```http
GET /a/changes/{id}/revisions/current/patch
```

`{id}` accepts:
- numeric (`4711`)
- `Project~Branch~Change-Id` (`myProject~master~I8473b9...`)
- raw `Change-Id` (422 if ambiguous across branches)

### 3) Post the review

```http
POST /a/changes/{id}/revisions/current/review
Content-Type: application/json
```

```json
{
  "message": "Automated review - 2 potential issues.",
  "tag": "autogenerated:review-bot",
  "notify": "OWNER",
  "omit_duplicate_comments": true,
  "comments": {
    "src/foo.c": [
      {"line": 42, "message": "Possible null-deref.", "unresolved": true}
    ]
  },
  "labels": {"Code-Review": 0}
}
```

**Bot hygiene (applies every time):**
- `tag: "autogenerated:..."` — UI collapses bot comments behind a toggle.
- `omit_duplicate_comments: true` — idempotent re-runs won't spam.
- `notify: "OWNER"` or `"NONE"` during testing — avoid email storms.
- Don't set a `Code-Review` vote unless the bot account has the permission (returns 400 otherwise). `Verified 0` or a plain `message` is safer default.

## Python httpx snippet (drop-in)

```python
import json, httpx

XSSI_PREFIX = b")]}'"

def _parse(resp: httpx.Response) -> dict | list:
    body = resp.content
    if body.startswith(XSSI_PREFIX):
        body = body.split(b"\n", 1)[1]
    return json.loads(body)

class GerritClient:
    def __init__(self, base: str, user: str, token: str):
        self._client = httpx.AsyncClient(
            base_url=base.rstrip("/"),
            auth=(user, token),
            headers={"Accept": "application/json"},
            timeout=30.0,
        )

    async def query_changes(self, q: str, opts: list[str], n: int = 50):
        r = await self._client.get(
            "/a/changes/",
            params=[("q", q), ("n", n), ("pp", "0"), *[("o", o) for o in opts]],
        )
        r.raise_for_status()
        return _parse(r)

    async def post_review(self, change_id: str, rev: str, review: dict):
        r = await self._client.post(
            f"/a/changes/{change_id}/revisions/{rev}/review",
            json=review,
        )
        r.raise_for_status()
        return _parse(r)
```

## Gotchas (ranked by how often they bite)

| # | Pitfall | Fix |
|---|---|---|
| 1 | `JSONDecodeError` on first call | Strip `)]}'` prefix |
| 2 | 403 on writes with valid creds | Missing `/a/` prefix |
| 3 | Duplicate bot comments after restart | `omit_duplicate_comments: true` |
| 4 | 400 on `labels.Code-Review` | Bot has no vote permission - drop the label |
| 5 | 422 on `{id}` | Raw `Change-Id` collides - use `Project~Branch~Change-Id` |
| 6 | Project names not found | Forgot to URL-encode `/` to `%2F` |
| 7 | Email spam during dev | `notify: "NONE"` |
| 8 | 429 | Exponential backoff; stagger poll workers |
| 9 | Missing fields in ChangeInfo | Add `o=CURRENT_REVISION` etc. - default response is minimal |
| 10 | Timezone drift in timestamps | Gerrit returns UTC `yyyy-mm-dd hh:mm:ss.fffffffff` - parse as UTC |

## Project mapping (gerrit_review_bot)

| Service | REST calls |
|---|---|
| **poller** (Python/httpx) | Owns all Gerrit I/O - `GET /a/changes/`, `GET .../diff/...`, `POST .../review` |
| **formatter** (Python, AI SDK) | No Gerrit calls; transforms diff to ReviewInput draft |
| **daemon** (Go) | Queue/worker; no Gerrit calls |

Keep Gerrit HTTP in the poller only - formatter/daemon stay provider-agnostic.
